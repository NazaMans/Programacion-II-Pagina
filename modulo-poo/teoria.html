Introducción a la Programación Orientada a Objetos (POO)

La Programación Orientada a Objetos (POO) es un paradigma de programación que utiliza "objetos" para diseñar aplicaciones y sistemas informáticos. Un objeto es una entidad que combina datos (atributos) y funciones que operan sobre esos datos (métodos).

Este enfoque permite modelar el mundo real de una manera más intuitiva, representando entidades como objetos que interactúan entre sí.

Ventajas de la POO

La POO ofrece múltiples beneficios en el desarrollo de software:

  • Reutilización de Código: Una vez que se define una clase, se puede instanciar varias veces o servir como base para otras clases.
  • Modificabilidad: Los cambios en una parte del sistema tienen un impacto menor en otras partes debido al encapsulamiento.
  • Facilidad para Detectar Errores: Los errores suelen estar confinados a los objetos individuales, facilitando su depuración.
  • Modularidad: El sistema se divide en módulos autocontenidos (objetos), lo que simplifica la gestión de proyectos grandes.
  • Flexibilidad (Polimorfismo): Permite que objetos de diferentes clases sean tratados de forma uniforme, lo que lleva a un código más adaptable.

¿Qué es un Objeto?

Un objeto es una entidad fundamental en la POO, que representa una abstracción o "cosa" con significado y límites claros en el dominio del problema.

Estructura de un Objeto:

  • Atributos (Estado): Son las características o propiedades que describen al objeto. Representan "lo que el objeto sabe".
  • Métodos (Comportamiento): Son las acciones que el objeto puede realizar o las operaciones que se pueden aplicar sobre él. Representan "lo que el objeto puede hacer".

Todo objeto posee:

  • Estado: Las propiedades del objeto en un momento dado, definidas por sus atributos.
  • Comportamiento: La forma en que un objeto actúa y reacciona, definido por sus métodos.
  • Identidad: Cada objeto es único y distinguible de otros, incluso si tiene el mismo estado y comportamiento.

Clases

Una clase es una descripción abstracta o un "molde" para un grupo de objetos con características y comportamientos similares. Define la estructura (atributos) y las funcionalidades (métodos) que tendrán todos los objetos creados a partir de ella.

Clases vs. Objetos:

  • La clase es una definición abstracta, un plano o un modelo.
  • El objeto es una instancia concreta de una clase. Se dice que se "instancia" un objeto a partir de una clase.

Por ejemplo, "Coche" sería una clase, mientras que "Mi coche rojo" o "El taxi de la esquina" serían objetos (instancias) de la clase Coche.

Terminología Clave en POO

Para comprender la POO, es esencial familiarizarse con los siguientes conceptos:

  • Abstracción:

    Proceso de identificar las características esenciales de una entidad y modelarlas en el software, ignorando los detalles no esenciales o irrelevantes para el problema en cuestión.

  • Encapsulamiento:

    Principio de agrupar los datos (atributos) y los métodos que operan sobre esos datos en una única unidad (la clase), y de restringir el acceso directo a los atributos desde fuera de la clase. Esto protege la integridad de los datos y simplifica la interfaz del objeto.

  • Instanciación:

    Es el proceso de crear un nuevo objeto (instancia) a partir de una clase. Cuando se instancia un objeto, se asigna memoria para él y se inicializa su estado.

  • Miembros Públicos, Privados y Protegidos:

    Estos son modificadores de acceso que controlan la visibilidad y el acceso a los atributos y métodos de una clase:

    • public: Los miembros son accesibles desde cualquier parte del programa. Son la interfaz pública de la clase.
    • private: Los miembros solo son accesibles desde dentro de la propia clase. Este es el modificador preferido para los atributos, fomentando el encapsulamiento.
    • protected: Los miembros son accesibles desde dentro de la propia clase y desde las clases que heredan de ella.
  • Constructores:

    Son métodos especiales de una clase que se invocan automáticamente cuando se crea un nuevo objeto (instancia) de esa clase. Se utilizan para inicializar los atributos del objeto. Un constructor tiene el mismo nombre que la clase y no tiene tipo de retorno.

    
class MiClase {
public:
    // Constructor por defecto
    MiClase() {
        // Inicialización
    }
    // Constructor con parámetros
    MiClase(int valor) {
        // Inicialización con valor
    }
};
  • Destructores:

    Son métodos especiales de una clase que se invocan automáticamente cuando un objeto de esa clase está a punto de ser destruido (sale de ámbito, se usa delete). Su propósito principal es liberar los recursos que el objeto haya podido adquirir durante su vida (como memoria dinámica). Un destructor tiene el mismo nombre que la clase, precedido por una tilde (~), no tiene tipo de retorno ni recibe parámetros.

    
class MiClase {
public:
    ~MiClase() {
        // Liberar recursos
    }
};
