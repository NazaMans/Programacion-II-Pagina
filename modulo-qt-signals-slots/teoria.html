Signal y Slot en Qt usando .ui

Introducción

En Qt, los Signals y Slots permiten conectar eventos de la interfaz con funciones que reaccionan a ellos. Cuando se usa Qt Designer y los archivos .ui, los widgets se definen visualmente y la conexión se realiza en el archivo .cpp del formulario correspondiente.

Video tutorial


Uso básico en widget.cpp

Supongamos que tenemos un botón llamado pushButton en nuestro formulario widget.ui. La conexión se realiza dentro del constructor del widget:


#include "widget.h"
#include "ui_widget.h"
#include <QDebug>

Widget::Widget(QWidget *parent)
    : QWidget(parent)
    , ui(new Ui::Widget)
{
    ui->setupUi(this);

    // Conectar señal clicked del botón a un slot personalizado
    connect(ui->pushButton, &QPushButton::clicked, this, &Widget::onPushButtonClicked);
}

// Slot definido en widget.h
void Widget::onPushButtonClicked() {
    qDebug() << "Botón presionado";
}

Nota: Qt Creator genera automáticamente la clase Ui::Widget desde el archivo .ui, permitiendo acceder a los widgets con ui->nombreWidget.

Uso de Signals y Slots personalizados

También podemos definir nuestros propios slots en la clase del widget:


class Widget : public QWidget {
    Q_OBJECT

public:
    Widget(QWidget *parent = nullptr);
    ~Widget();

public slots:
    void mostrarMensaje(const QString &texto); // slot personalizado
};

// Conexión en widget.cpp
connect(this, &QPushBotton::Click, this, &Widget::mostrarMensaje);

Resumen

  • Para widgets creados con .ui, usamos ui->widgetName para acceder a ellos.
  • Las señales se conectan a slots usando connect() dentro del constructor.
  • Se pueden crear slots personalizados dentro de la clase.
